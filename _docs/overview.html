Introduction

Hey there! You're probably reading this documentation site because you want to do something related with the Student Server Space, and that sounds awesome! Here, we have pages that tell you how to submit a project, deploy a project, manage the server, and everything in between. Plus, you'll find a set of policies and guidelines that make clear what's allowed on the Student Server Space, and what isn't. You can find all of this information on your sidebar. We'll also answer some common questions about SSS.

What is SSS?

The Student Server Space is a set of resources and a community to help student developers grow. In particular, there are two features that are discussed in this documentation. Firstly, student developers can submit a project to SSS for it to be featured on the SSS website. This gives projects more exposure, and can also invite others to help develop the project. Secondly, student developers can deploy a project to an SSS webserver. This allows students to get a domain and hosting for free, which is often expensive for students to accquire. To do either of these actions requires a set of restrictions, which is outlined in our code and content policies.

How does it work?

If you want to use SSS, I suggest you check out our submission and deployment guides, which walk you through the process of making your own projects!

If you're just wondering about the technology behind SSS, we'll get back to you on that.

Something's wrong!

We're human, and we make mistakes too! If you think something's wrong with our documentation, or with our website in general, send us an issue on GitHub.

I'd like to help!

Awesome! You can check out our GitHub organisation to see all of the tools we use for this system, or work on this website's repository. Once you've finished some work, send us a pull request, and we'll look over your changes, and either approve them, or give you feedback.
