Notice

This article tells you how to deploy your project to the Student Server Space, not how to submit it. To submit your code, first read our Submission Guide and submit your code, and then head back here to deploy it.

To clarify, submitted projects do not need to be deployed (they can be hosted externally).

Overview

This guide walks you through deployment on Student Server Space, or how to get your project up and running on our webserver.

Requirements

Before you deploy your project to Student Server Space, make sure that you've checked off everything in our Content Policy, and our Code Policy. If part of your project doesn't adhere to these guidelines, we cannot accept it for deployment.

In addition, we need your project to contain all the files necessary for deployment: this repository contains an example of each file required, and a detailed description of each.

Finally, check that your project is supported by Student Server Space. As we're still developing a lot of the functionality in SSS, it's possible that your project isn't supported yet. Once you've checked off all of these things, you're ready to go!
